Gmail Setup Guide for Gmail Manager

A practical step-by-step setup guide for connecting the MVP to Gmail in read-only mode and writing action items to one Google Sheet.

Setup Checklist

  1. Confirm prerequisites
  2. Use the quick setup path
  3. Create a Google Cloud project
  4. Enable Gmail API and Google Sheets API
  5. Configure OAuth consent
  6. Create Desktop OAuth credentials
  7. Download credentials.json
  8. Create or choose a Google Sheet
  9. Configure the local project
  10. Run sample mode
  11. Run Gmail mode
  12. Verify the Sheet output
  13. Troubleshoot common issues

1. Confirm Prerequisites

Before starting, make sure you have:

  • A Google account with Gmail enabled.
  • Access to Google Cloud Console.
  • Python 3.11 or newer installed locally.
  • The Gmail Manager project on your machine.
Security intent This project reads Gmail only. It does not send, delete, archive, label, or modify messages inside Gmail.

2. Quick Setup Path

This quick path is adapted from the local SpendLens setup notes and adjusted for Gmail Manager.

  1. Go to console.cloud.google.com.
  2. Click Select a project, then New Project.
  3. Name the project Gmail Manager.
  4. Enable the Gmail API.
  5. Enable the Google Sheets API.
  6. Create an OAuth 2.0 Client ID.
  7. Choose application type Desktop app.
  8. Name it Gmail Manager Local.
  9. Download the OAuth JSON file.
  10. Rename it to credentials.json.
  11. Place it in the project root folder.
  12. Add your Gmail address as a Test user if the app is in External testing mode.
  13. Run Gmail mode and approve read-only Gmail access in the browser.
SpendLens detail carried over The important OAuth pattern is the same: Desktop app credentials, your Gmail address added as a test user, and browser approval for read-only Gmail access.

3. Create a Google Cloud Project

  1. Open Google Cloud Console .
  2. Click the project selector near the top of the page.
  3. Click New Project.
  4. Use a clear name, such as Gmail Manager.
  5. Click Create.
  6. Make sure the new project is selected before continuing.

4. Enable the Required APIs

The app needs two Google APIs:

Gmail API

Used to read recent Gmail messages in read-only mode.

Google Sheets API

Used to create/update tabs and append action rows to the Sheet.

  1. In Google Cloud Console, open APIs & Services.
  2. Click Library.
  3. Search for Gmail API.
  4. Open it and click Enable.
  5. Return to the API Library.
  6. Search for Google Sheets API.
  7. Open it and click Enable.

5. Configure the OAuth Consent Screen

OAuth consent tells Google what app is requesting access and what the user is approving.

  1. Open Google Auth Platform or APIs & Services > OAuth consent screen.
  2. If prompted, click Get Started.
  3. For app name, enter Gmail Manager.
  4. Choose your user support email.
  5. For audience/user type:
    • Choose Internal if this is only for your Google Workspace organization.
    • Choose External if you are using a personal Gmail account or testing outside a Workspace organization.
  6. Add your developer contact email.
  7. Save and continue.
  8. If using External mode, add your own Gmail address as a test user.
Important for External apps Keep the app in testing mode for this MVP. Add only the Gmail accounts that should test the tool.

6. Create Desktop OAuth Credentials

  1. Open APIs & Services > Credentials.
  2. Click Create Credentials.
  3. Select OAuth client ID.
  4. For application type, choose Desktop app.
  5. Name it Gmail Manager Local.
  6. Click Create.
  7. Click Download JSON.

Rename the downloaded file to credentials.json.

7. Place credentials.json in the Project

Move credentials.json into the project root:

gmail-to-action-sheet/
  credentials.json
  src/
  data/
  tests/
Do not commit credentials credentials.json, token.json, and .env are intentionally ignored by git.

8. Create or Choose a Google Sheet

  1. Open Google Sheets.
  2. Create a blank spreadsheet.
  3. Name it Gmail Manager.
  4. Copy the Sheet ID from the URL.

The Sheet ID is the long value between /d/ and /edit:

https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID_HERE/edit

The script will create or update these tabs:

  • Today Action List
  • Processed Emails
  • Settings

9. Configure the Local Project

From the project root, create a virtual environment and install dependencies:

cd gmail-to-action-sheet
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Create your local environment file:

cp .env.example .env

Edit .env and set your Sheet ID if you want a default:

GOOGLE_CREDENTIALS_FILE=credentials.json
GOOGLE_TOKEN_FILE=token.json
GOOGLE_SHEET_ID=YOUR_SHEET_ID_HERE
PROCESSED_LOG_FILE=processed_emails.csv
OUTPUT_CSV_FILE=output_action_list.csv

10. Test Sample Mode First

Sample mode verifies the classifier without Gmail or Google credentials.

python src/main.py --mode sample

To also write a local CSV output:

python src/main.py --mode sample --output-csv

You should see classified action items printed in the terminal.

11. Run Gmail Mode

Use the Sheet ID from .env:

python src/main.py --mode gmail --days 1

Or pass the Sheet ID directly:

python src/main.py --mode gmail --days 1 --sheet-id YOUR_SHEET_ID_HERE

The first run opens a Google OAuth browser consent flow. Choose the Gmail account you want to read from and approve the requested access.

12. Verify the Requested Permissions

The MVP uses these scopes:

Scope Purpose
https://www.googleapis.com/auth/gmail.readonly Read Gmail messages only. No sending, deleting, labels, or archiving.
https://www.googleapis.com/auth/spreadsheets Write action rows into the configured Google Sheet.

13. Confirm the Google Sheet Output

After Gmail mode runs, open your spreadsheet and check:

  • Today Action List contains new action rows.
  • Processed Emails contains Gmail message IDs already handled.
  • Settings contains the default MVP settings.

Run the same command again. Existing Gmail message IDs should be skipped.

14. Troubleshooting

Access blocked or app not verified

If your app is in External testing mode, add your Gmail address as a test user in the OAuth consent screen settings.

Gmail connect fails

This was also called out in the SpendLens setup notes. The most common fix is to confirm your email address is added under Test users in Google Cloud Console.

Missing credentials.json

Confirm that credentials.json is in the project root and the value of GOOGLE_CREDENTIALS_FILE matches the filename.

Wrong Google account selected

Delete token.json and run Gmail mode again. The browser consent flow will ask you to choose an account again.

Changed OAuth scopes

Delete token.json after changing scopes so Google can issue a new token with the updated permissions.

Sheet not updating

Confirm the Sheet ID is correct and that the Google account used in the OAuth flow can access the spreadsheet.
